# Запуск
Чтобы запустить сервер нужно вызвать `docker-compose up`. Создастся 2 сервиса: nuts и app

# Эндпоинты
| Метод | Эндпоинт                 | Описание                                           |
| ----- | ------------------------ | -------------------------------------------------- |
| GET   | /api/todos/              | Получить все задачи                                |
| GET   | /api/todos/{item_id}     | Получить задачу по ID                              |
| POST  | /api/todos/              | Создать новую задачу                               |
| PUT   | /api/todos/{item_id}     | Обновить задачу по ID                              |
| DELETE| /api/todos/{item_id}     | Удалить задачу по ID                               |
| POST  | /api/todos/force-fetch   | Принудительно вызвать фоновую задачу               |
| GET   | /api/todos/health        | Проверить соединение с NATS                        |
| WS    | /ws/tasks                | WebSocket-канал для уведомлений в реальном времени |

# Описание

API для TODO, загружает задачи с https://dummyjson.com/todos, хранит id пользователя, к которому относится задача, заголовок, описание, даты создания и изменения, выполнена ли задача и внешний id.

Сообщения из NATS вызывают broadcacst, поэтому в ws отправляется 2 сообщения: одно напрямую, второе вызывается через NATS.
